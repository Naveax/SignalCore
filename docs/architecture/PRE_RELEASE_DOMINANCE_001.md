# SignalCore v0.0.1 Pre-Release Dominance Core

SignalCore is version-locked to **0.0.1** until the repository owner explicitly authorizes a version change. The release channel remains **pre-release / pre-alpha**.

This package consolidates the previously planned P0–P2.3 and V0.7–V1.0 work into one release identity rather than pretending that separately numbered milestones were independently shipped.

## P0: zero-friction product surface

- One-command plan-first, backup-first installation.
- `install`, `wrap`, `doctor`, `stats`, `upgrade`, and `repair` commands.
- Ten native provider contracts, fifteen framework contracts, and eighteen host contracts.
- Automatic host configuration target of at least fourteen hosts.
- Version-locked upgrade policy; `0.0.2` and later are rejected until owner authorization.
- Live certification is receipt-bound. Internal contracts are not mislabeled as live external integrations.

## P1: coding and long-context dominance foundation

### SignalBench 2.0

- 150 coding-task slots across nine task categories and ten languages.
- Six isolated arms: plain baseline, SignalCore, Token Savior, Context Mode, Headroom, and Volt/LCM.
- Thirty deterministic paired repetitions.
- Identical model, provider, reasoning, context-window, permission, verifier, and repository-tree requirements.
- Failed work receives zero credit.
- Synthetic or unmaterialized corpus slots cannot open a public superiority claim.

### Structural Intelligence V2

- Multi-graph nodes and exact-evidence edges.
- Calls, imports, inheritance, implementation, override, data-flow, taint-flow, tests, build targets, dependencies, stack traces, ownership, change-frequency, and rename history.
- Query-adaptive lexical/path/ownership/centrality/change ranking.
- Reverse impact and affected-test traversal.

### Unbounded context

“Infinite context” does not mean an infinite model token window. It means:

1. history grows in exact external storage without forced session restart;
2. the active model window remains strictly bounded;
3. recursive summaries and exact evidence references preserve recoverability;
4. current temporal truth outranks superseded state;
5. map/reduce workers retain per-task and global provenance.

The committed stress harness covers 32K, 64K, 128K, 256K, 512K, 1M, 2M, and 10M virtual history tokens. Every tier must remain within the active budget, preserve exact references, and report `forced_restart=false`.

## Claim boundaries

- `EXTERNAL_SUPERIORITY_NOT_PROVEN` remains active until the live external-arm gate passes.
- `PUBLIC_PRODUCT_MATURITY_NOT_PROVEN` remains active until the real beta gate passes.
- `UNBOUNDED_EXTERNAL_HISTORY_WITH_BOUNDED_ACTIVE_WINDOW` is the precise context claim.

## Version policy

The following must remain synchronized at `0.0.1`:

- `VERSION`
- Python package version
- Python runtime `__version__`
- TypeScript package version
- skill metadata
- marketplace/extension metadata
- CodeMeta
- README/status output

The package is always published as a pre-release until the owner explicitly changes this policy.

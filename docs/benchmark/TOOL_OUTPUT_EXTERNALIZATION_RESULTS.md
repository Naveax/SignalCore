# Tool-Output Externalization V2 — Internal Results

## Boundary

The committed benchmark is deterministic and internal. No competitor executable, coding model or provider billing arm is run. Status remains `SUPERIORITY_NOT_PROVEN`.

## Fixture composition

The corpus includes:

- 180-file source diff;
- 4,000-test output with a failure needle;
- 25,000-line service log with a secret-bearing fatal event;
- 9,000-record JSON array with a rare error;
- 18,000 search results;
- 600 KiB binary payload;
- exact duplicate capture;
- append-only delta capture.

## Development result

On the local development run, all three profiles passed:

- first-view quality gate;
- byte-exact reconstruction;
- artifact and segment hash validation;
- Merkle proof validation;
- rare-needle retrieval;
- duplicate suppression;
- delta-lineage checks.

Approximate development-run totals:

| Profile | Raw bytes | Visible reduction | Runtime |
|---|---:|---:|---:|
| compact | 4,308,273 | 99.9149% | 2.65 s |
| balanced | 4,308,273 | 99.9150% | 2.62 s |
| audit | 4,308,273 | 99.9150% | 2.92 s |

Runtime values are environment-specific and are not suitable for external marketing claims. The committed JSON receipt generated by CI is authoritative for the branch.

## Performance defect found and fixed

The first implementation created one semantic segment per `file:line` search result. The 18,000-result fixture therefore produced 18,000 segments and approximately 21.5 seconds of retrieval latency.

The final implementation groups search output by target byte size. The full multi-fixture benchmark then completed in approximately 2.5–2.9 seconds per profile on the development environment while retaining the rare-result retrieval gate.

## Required external proof

A superiority claim still requires:

1. identical repository tasks;
2. identical model, reasoning and context window;
3. executable competitor arms;
4. provider usage receipts;
5. equal or higher verified task pass rate;
6. failure-inclusive quota accounting;
7. repeated cold and warm runs.

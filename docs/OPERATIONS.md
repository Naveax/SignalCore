# Syntavra Operations

Supported operator surface:

```bash
syntavra setup
syntavra setup --apply
syntavra setup --repair --apply
syntavra status
syntavra status --doctor
syntavra status --savings
syntavra status --profile
syntavra status --memory
syntavra status --evidence
syntavra run
syntavra prove
```

Installation and updates are checksum-verified, atomic and rollback-capable. Headless jobs emit deterministic status, artifacts and receipts. Compatibility commands remain executable where required but are not part of onboarding.

Token reporting separates provider-observed usage from locally tokenized or estimated source attribution. Missing provider data remains unknown rather than being inferred as billing truth.
